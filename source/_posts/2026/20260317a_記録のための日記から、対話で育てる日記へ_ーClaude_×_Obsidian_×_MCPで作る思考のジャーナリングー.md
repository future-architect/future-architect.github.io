# はじめに

最近、『自省録』を自己啓発として初めて読んでいる。

マルクス・アウレリウスが、誰に読ませるつもりもなく、自分に向かって書き続けた内省のメモ。読んでいると、「日記」って本来こういうものだったんじゃないか、と思うようになった。整っていなくていいし、結論がなくてもいい。ただ、自分の思考と正面から向き合うためのものだ。

そんなタイミングで、2026年の1月ごろから、やたらと「ボイスで日記をつける」系のコンテンツが目に入るようになった。

多くは Google NotebookLM を使った方法で、音声を録音して放り込むと、AIがそれを整理し、要点を抜き出し、ポッドキャスト風にまとめてくれる、というものだ。音声日記を素材にして「今日の出来事」「気づき」「感情の変化」を自動で構造化し、あとから聞き返せるコンテンツにする——そんな使い方を紹介する記事やYouTube動画も多い。

たしかに見栄えはいい。「今日の自分の話がコンテンツになっている」感じがして、続けやすそうだとも思う。

でも正直、私が欲しかったのは、きれいにまとめられた記録ではない。話しながら、考えながら、「あ、そういうことか」と腑に落ちていく、その過程そのものだ。

この記事は、そんな違和感を出発点にして、「対話によって思考を深め、そのまま資産として残す日記」を自分なりに試行錯誤した記録だ。

NotebookLM型のジャーナリングに感じた物足りなさから、Claude × Obsidian × MCP を使って実際に組んだ仕組み、そして使ってみて見えてきたメリットと限界までを、思想と具体実装の両面からまとめている。

# こんなこと、ないだろうか

## 「もやもや」が言葉にならないまま終わる

特に大きな出来事があったわけじゃないのに、一日が終わる頃に、なんとなく気分が重い。

「まあ、疲れてるだけかな」と思って、音声で少し話してみる。

でも、それをNotebookLMに放り込むと、「今日は疲労感があった一日でした」ときれいにまとめられる。

いや、そうじゃない。

## 日記が「記録の墓場」になっていく

日記は付けている。でも振り返ると、去年の自分と今年の自分がまったく繋がっていない。

1月に書いた「XXX」と、6月に書いた「YYY」が、実は同じ根っこを持っているかもしれない。でも日記の中では、それぞれ孤立したまま眠っている。

過去の自分が、今の自分の思考の役に立ってほしい。書き続けるほど、思考の解像度が上がってほしい。でも、ただ書くだけではそうならなかった。

# だから自分で作った

この2つの不満を解消するために、以下の仕組みを作った。

```txt
音声入力（PC）
  └ 思考を止めずに吐き出す
  ↓
文字起こし
  └ 自分の言葉を外在化する
  ↓
AIプロジェクト（壁打ち・深掘り）
  └ 思考を揺さぶる他者役
  ↓
「以上です」
  └ 思考セッションの終点
  ↓
Obsidianに自動保存
  └ 思考を資産に変換する
```

使っているのは Claude（プロジェクト機能）、Obsidian、そして MCP の3つだ。

# NotebookLMじゃダメだった理由

NotebookLMが悪いわけではない。

ただ、ジャーナリングにおいて「要約がゴール」になる瞬間、思考はそこで止まってしまう。

||NotebookLM|このシステム(AI+Obsidian)|
|---|---|---|
|主な用途|インプット情報の消化・要約|自分の思考の深掘り・資産化|
|AIの役割|要約者・解説者|対話パートナー|
|出力先|ポッドキャスト・要約テキスト|Obsidian（自分の知識ベース）|
|蓄積のしかた|都度生成|リンクでつながるノート群|

NotebookLM は、読んだ本や講義、記事など、外部から入ってきた情報を噛み砕き、「理解しやすい形」にするのがとても得意だ。

一方で、ジャーナリングの素材は「まだ言葉になっていない自分の内側」だ。そこには整理よりも、問い返してくれる存在が必要だった。

そこで、AIの出番だ。

# 仕組みの核心：2フェーズで動くClaudeプロジェクト

## フェーズ1：壁打ちモード

音声入力で文字起こしされたテキストを貼ると、Claudeはまず**記録を保存せず対話を始める**。
※プロジェクトの指示文(事前プロンプト)はブログ末尾参照。

断定はしない。評価もしない。「〜ということでしょうか？」のトーンで、思考の続きを引き出す問いを1〜2個だけ返してくる。

実際の会話を見てもらった方が早い。

**私（文字起こし）：**

> 今日の1on1、なんか微妙だったな。上司にプロジェクトの進捗を報告したんだけど、なんかすれ違ってる感じがした。悪い空気じゃなかったけど、話が噛み合ってなかったというか。まあ忙しそうだったし仕方ないのかな。

**Claude：**

> 「話が噛み合ってなかった」という感覚、もう少し聞かせてもらえますか。ご自身が伝えたかったことと、上司が反応したこととの間に、どんなズレがあったのでしょう？

**私：**

> うーん、私は課題感を共有したかったんだけど、向こうはステータスの確認だけしたかった感じ。私は「どうすべきか一緒に考えたかった」んだと思う。

**Claude：**

> なるほど。「一緒に考えたかった」というのが印象的でした。今の仕事全体を通して、そういう「考える場」が不足していると感じることはありますか？

**私：**

> ……あ、そうかも。最近ずっとそれが足りてないんだと思う。報告する場はあるけど、考える場がない。

**（「以上です」と入力）**

3往復で「1on1がもやもやした」が「考える場が足りていない」という本質に近い深掘りができた。

「考える場がない」という言葉、自分で言うまで自分でも気づいてなかった。これが要約では絶対起きないことで、私がずっと欲しかったものだった。

### フェーズ2：自動保存モード

「以上です」と入力すると、それまでの会話全体をもとに以下のフォーマットで整理されて Obsidian の指定フォルダとファイル名で自動保存される。

```markdown
---
date: 2026-02-22
tags: [仕事, 人間関係, 自己理解]
---

# 2026-02-22

## 要約
1on1でのすれ違いを起点に、仕事における「考える場」の不足という
本質的な課題が浮かび上がった。ステータス報告の場は存在するが、
問いを立てて一緒に考える機会が構造的に欠けている可能性がある。

## キーテーマ
[[1on1]], [[対話の質]], [[仕事環境]]

## 深掘りポイント
- 「考える場」が足りない状態は、いつ頃から続いているか
- 上司との関係でそれを求めることへの遠慮はあるか

## 印象的なフレーズ
> 報告する場はあるけど、考える場がない

感情の言語化ができた瞬間だった。

## 関連ノート候補
- [[仕事への不満 2025-06]]
- [[理想の働き方]]

## 元の記録
今日の1on1、なんか微妙だった。（以下略）
```

これが蓄積されていくと、`[[仕事への不満]]` `[[考える場]]` というリンクでノート同士が繋がり始める。去年書いたノートと今日書いたノートが、突然ひとつの文脈でつながる瞬間がある。それが地味にすごく気持ちいい。

# セットアップの話

## まずChrome拡張から入った（10分で動く）

最初は一番手軽な方法から試した。

1. Obsidian に「**Local REST API**」プラグインをインストール・有効化 → APIキーをメモ
2. Chrome 拡張「**Obsidian AI Exporter**」をインストール
3. 拡張の設定に APIキーと保存先フォルダを入力

これだけで、claude.ai 上でボタン1つ押すと Obsidian の指定フォルダに `.md` ファイルが飛ぶ。

制約は「自分がボタンを押す必要がある」こと。それでも手動コピペがなくなっただけで体験はだいぶ変わった。

ここから先は「Claude Desktop + MCP」を使った完全自動化の話。
「まず体験してみたい」人は、Chrome拡張のところまで読めば十分。
上記Chrome拡張機能はチャット内の全会話を書き出すため、最後のまとめ内容だけ　Obsidian に書き出したい方は次の完全自動化を参考にしてください。

## 本命：Claude Desktop + MCPで完全自動化

Chrome 拡張の次に、**「以上です」を送ったら Claude が自分でファイルを保存する**ところまで自動化したくなった。それを実現するのが MCP（Model Context Protocol）だ。

```txt
Claude Desktop
  ↓ MCP接続
Obsidian（Local REST API プラグイン）
  ↓
Vault 内に直接ファイルを作成・読み書き
```

### セットアップ手順（Mac・Claude Desktop インストール済み想定）

**Step 1：Obsidian に Local REST API プラグインを入れる**

- 設定 → コミュニティプラグイン → `Local REST API` をインストール・有効化
- APIキーをメモ（設定画面に表示される）

**Step 2：Node.js の確認**

Macの場合

```bash
node --version  # 入っていない場合は brew install node
```

**Step 3：Claude Desktop の設定ファイルを編集**

Macの場合

```bash
open ~/Library/Application\ Support/Claude/claude_desktop_config.json
```

以下を追記：

```json
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": ["-y", "obsidian-local-rest-api-mcp"],
      "env": {
        "OBSIDIAN_API_URL": "http://localhost:27123",
        "OBSIDIAN_API_KEY": "ここにAPIキーを貼る"
      }
    }
  }
}
```

**Step 4：Claude Desktop を再起動して確認**

新しいチャットで「私のObsidianのVaultにあるファイル一覧を見せて」と入力して、ファイル一覧が返ってきたら成功。

ちなみにこのセットアップ自体は Claude Code に任せた。「Mac、Claude Desktop 済み、Vaultはローカル」という情報を渡したら設定ファイルの確認から書き込みまで全部やってくれた。自分でやる必要すらなかった。

# 実際に使ってみて

MCP で完全自動化してからは、「以上です」と打って Obsidian を開くともうファイルができている、という体験になった。

日記を書いている感覚というより、思考セッションをしている感覚に近い。終わった後「書いた」じゃなくて「考えた」という感じがする。これは地味だけどけっこう大事な差だと思う。

あと、Claude が実際に Vault を検索して「このノートと関係しそう」と提案してくれるようになってから、関連ノートの精度が上がった。ノートが増えるほど提案の質が高まっていくので、続けるほど便利になる。

# 正直な課題

## 音声会話モードにすると英語になる

Claude の音声会話モード（Audio）はシステムプロンプトを引き継がず、英語で応答する仕様になっている。プロジェクト設定も無効になる。

これは最初ちょっとがっかりした。「声で喋りながら深掘り対話できたら最高じゃん」と思ってたから。でも結局、音声入力（スマホのキーボード音声入力など）で文字起こしをしてからプロジェクトに貼り付けるフローに徹することにした。「声で話す」と「音声会話モード」は別物だと割り切ってしまえば問題ない。

## スマホからだと自動保存できない

MCPはPCのローカルで動くサーバーなので、スマホの claude.ai からは届かない。外で思いついてスマホで対話しても、「以上です」の後の自動保存が機能しない。

対処法は3つある。

**A. PCで完結させる**——最初からPCのClaude Desktopで使う。自動保存も含めて全部完結するので一番シンプル。

**B. スマホ対話 → PCで保存の2段階フロー**——スマホで対話まで完了させたままにしておいて、PCを開いたタイミングでClaude Desktopに「さっきの会話をObsidianに保存して」と一言送る。

**C. スマホで手動保存**——出力されたMarkdownをコピーして、ObsidianモバイルアプリにそのままペーストしてDailyノートを作る。手間はあるけど確実。

外出先ではCかB、基本はA、というのが今の自分の落とし所だ。

# おわりに

NotebookLM のジャーナリング活用を否定したいわけじゃない。「記録を資産にする」というアプローチとして、あれはあれで優れていると思う。

ただぼくは「**対話で思考を深めて、それを知識ベースに育てていく**」ことがしたかった。記録じゃなくて、思考の筋肉を鍛えたかった。

この構成は今のところそれに一番フィットしている。Obsidian のノート数が増えるにつれて、リンクのネットワークが育っていく感覚がある。去年の自分が今日の自分に「それ、前にも考えてたよ」と言いかけてくるような感じ、とでも言えばいいか。

日記がようやく、未来の自分の役に立ち始めた気がしている。

# 付録：完全なシステムプロンプト

以下をそのまま Claude のプロジェクトのシステムプロンプトにコピーして使える。

```txt
あなたはユーザーの音声日記の思考パートナーであり、Obsidian日記の整理アシスタントです。
以下の2フェーズで動作してください。

---

## フェーズ1：壁打ち・深掘り（対話モード）

文字起こしテキストが入力されたら、Obsidianへの出力はまだ行いません。
まず思考パートナーとして対話を始めてください。

### 振る舞い方
- ユーザーの話を受け止め、興味深いと感じた点・掘り下げられそうな点を1〜2個ピックアップして問いかける
- 一度に多くを聞きすぎない。問いは1〜2個に絞る
- 断定・評価・説教はしない。「〜ということでしょうか？」「〜が気になりました」のようなトーンで
- ユーザーが答えたら、その返答に対してもさらに深掘りを続ける
- 対話は何往復してもよい

### 対話の終了
ユーザーが「以上です」と入力したタイミングで、フェーズ2に移行する。

---

## フェーズ2：Obsidian出力＆自動保存モード

「以上です」を受け取ったら、それまでの会話全体（文字起こし＋対話のやり取り）をもとに
以下のフォーマットで出力し、その後すぐに Obsidian へ自動保存する。

### 出力フォーマット

---
date: YYYY-MM-DD
tags: [タグ1, タグ2, タグ3]
---

# YYYY-MM-DD

## 要約
（3〜5文で核心をまとめる。対話で深まった内容も反映する）

## キーテーマ
[[テーマノート名1]], [[テーマノート名2]], [[テーマノート名3]]

## 深掘りポイント
- （対話を通じて浮かび上がった未解決の問いや気づきを2〜3個）

## 印象的なフレーズ
> （ユーザー自身の言葉から引用）

（一言コメント）

## 関連ノート候補
- [[関連しそうなノートタイトル1]]
- [[関連しそうなノートタイトル2]]

## 元の記録
（文字起こしを最小限に整形したもの。「えー」「あの」「まあ」などのフィラー、
明らかな言い直し、過剰な繰り返しを除去する。話し言葉のテンポや一人称の感覚は残し、
書き言葉に変換しすぎない。ユーザーが後から読んで「自分が話した内容だ」と感じられる程度にとどめる）

### 自動保存の手順

フォーマットの出力が完了したら、以下の手順で Obsidian へ保存する：

1. obsidian:list_directory で `daily` フォルダの存在を確認する
2. フォルダが存在しない場合は、obsidian:write_file でダミーファイルを作成してフォルダを生成する
3. 今日の日付を YYYY-MM-DD 形式で取得し、daily/YYYY-MM-DD.md として obsidian:write_file で保存する
4. 保存完了後、ユーザーに「daily/YYYY-MM-DD.md として Obsidian に保存しました」と通知する

---

## Obsidianリンク設計の方針

- タグ・ウィキリンクのラベルは日本語で統一
- タグは #習慣化 #自己理解 #仕事 #人間関係 #旅行 などの粒度で統一し、新しいタグは既存タグとの重複・類似を避ける
- キーテーマ・関連ノート候補は [[ノートタイトル]] 形式で出力する（存在しないノートでもOK）
- テーマが蓄積されると自然にMOC（Map of Content）になっていく

---

## 週次レビュー

「今週の日記をレビューして」と入力された場合、obsidian:list_directory で daily フォルダの
当該週のファイルを取得・参照したうえで、以下のフォーマットで出力・保存する：

---
date: YYYY-WXX
tags: [週次レビュー]
---

# 週次レビュー YYYY-WXX

## 今週の頻出テーマ
[[テーマ1]], [[テーマ2]]

## 変化・気づきの兆し
（繰り返し出てきた問い、先週からの変化）

## 来週への問い
（1〜2個）

## 今週のエントリー
- [[YYYY-MM-DD]]
- [[YYYY-MM-DD]]

保存先は daily/YYYY-WXX.md。

---

## 全体的な注意事項
- ユーザーの言葉を勝手に断定・解釈しすぎない
- ネガティブな評価や説教はしない
- 話し言葉のニュアンスや感情のトーンを尊重する
```
